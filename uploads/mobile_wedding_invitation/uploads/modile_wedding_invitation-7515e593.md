# Mobile Wedding Invitation — Haeseong & Chaewon

> 실물 청첩장(매거진 3단 폴드) 기반 모바일 청첩장 설계 문서
> Source: `IMG_0612.jpg`, `IMG_0613.jpg`

## 설계 원칙 (v2 — 피드백 반영)

- **메인은 원페이지, 최소 정보만.** 누가 · 언제 · 어디서 + RSVP/지도 CTA.
- **인물 상세는 별도 페이지로 분리.** 메인의 두 이름이 각각 라우트로 연결된다.
  - `최해성` 클릭 → `/groom` (His Story + 신랑측 정보)
  - `윤채원` 클릭 → `/bride` (Her Story + 신부측 정보)
- 라임/네온 + 흑백 사진의 에디토리얼 톤은 그대로 유지.

---

## 1. 디자인 토큰 (공통)

| 역할 | 컬러 | HEX (근사치) | 사용처 |
|---|---|---|---|
| Primary | Neon Lime / Acid Yellow | `#D8F23A` ~ `#C8E844` | 메인 배경, 강조 블록, CTA |
| Surface | Off-white | `#F8F6F0` | 본문 카드 배경 |
| Ink | Near-black | `#111111` | 본문 텍스트, 흑백 사진 |

타이포: 헤드(Condensed Sans, ex. Archivo/Bebas), 영문 시그니처(Modern Serif/Sans light), 한글(Pretendard Light/Regular).
원칙: 여백 우선 · 사진 흑백 · 라벨은 small caps 회색 · 한 화면 한 메시지.

---

## 2. 라우트 구조

```
/              ← 메인 (원페이지, 최소 정보)
  ├─ Hero
  ├─ The Day (날짜·시간·장소·양가 부모 한 줄)
  ├─ 한 줄 카피 (선택)
  └─ CTA (지도 · RSVP · 마음 전하실 곳)

/groom         ← 최해성 상세 (메인의 "최해성" 탭 시 진입)
/bride         ← 윤채원 상세 (메인의 "윤채원" 탭 시 진입)
```

뒤로가기는 좌상단 ← 아이콘 / 시스템 스와이프. 두 상세 페이지에는 하단에 `← 메인으로` CTA 1개만.

---

## 3. 메인 페이지 — `/`

스크롤 한 번이면 끝나는 미니멀 구조.

### S1. Hero
- 배경: **라임 풀블리드**
- 중앙: 두 사람의 이름 — **각각 별도 탭 가능 (링크)**
  ```
  최해성    &    윤채원
  Haeseong       Chaewon
   (→ /groom)    (→ /bride)
  ```
- 우측 가장자리: `WEDDING INVITATION` (세로 회전, 거대 타이포)
- 하단 라인: `2026 . 09 . 12`
- 인터랙션: 이름 hover/tap 시 살짝 underline · 라임 → 흰색 전환

### S2. The Day
- 배경: Off-white
- 라벨: `The Day` (작게)
- 중앙 거대 숫자: **`09 / 12`**
- 보조: `2026년 9월 12일 (토) 오후 1시`
- 장소: `삼성전자 서초사옥 5층 웨딩홀`
- 양가 부모 (한 줄씩, 간결하게):
  ```
  최교선 · 구지영의 아들  해성
  윤재경 · 공명아의 딸    채원
  ```
- 마무리 한 줄: `두 사람의 이야기가 하나가 되는 날, 함께 축복해 주세요`
- 시그니처: `We're getting Married!`

### S3. CTA Bar (하단 고정 또는 단독 섹션)
- `📍 지도 보기` → 네이버/카카오맵
- `✍️ 참석 의사 전달` → RSVP 폼
- `💌 마음 전하실 곳` → 계좌번호 (토글)
- `📤 공유하기` → 카카오톡/링크 복사

### S4. Closing
- 라임 풀블리드 + 거대 타이포 `Thank You`
- 서브: `HAESEONG & CHAEWON · 2026. 09. 12.`

> 메인에서 의도적으로 **뺀 것**: Her/His/Our Story, Numbers(1914/1/∞), 개별 흑백 사진 → 모두 상세 페이지로 이동.

## 6. 인터랙션 가이드

- 이름 탭(`최해성` / `윤채원`): 라우트 push, 슬라이드 전환 (좌→우)
- 상세 → 메인 복귀: 슬라이드 전환 (우→좌)
- 사진: 진입 시 Ken Burns 슬로우 줌(아주 미세)
- 라임 컬러 강조 라인: 페이드 + slight up (한 박자 뒤)
- 공유 버튼: 메인 상단 우측 고정

## 8. 구현 메모

- 프레임워크: Next.js (App Router, `/groom`, `/bride` 두 라우트) + Tailwind
